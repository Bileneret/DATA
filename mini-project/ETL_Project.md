# Варіант 14. Агент інтеграції даних

### [Бекенд (main.py)](main.py) | [Логіка агента (agent.py)](agent.py)

#### Опис:
Проєкт реалізує автономного ШІ-агента на базі архітектури ReAct (Reason + Act) із використанням мовної моделі Gemini (`gemini-2.5-flash`) та фреймворку LangChain (Tool Calling Agent). Весь процес побудований на класичних етапах ETL (Extract, Transform, Load). Агент самостійно отримує високорівневу мету у вигляді промпту, аналізує її, розбиває на підзадачі та послідовно викликає задекларовані Python-інструменти (Tools) для збору даних з різних джерел, їх очищення, трансформації та фінального збереження в сховище. Система розгорнута як Full-stack застосунок із бекендом на FastAPI та інтерактивним веб-інтерфейсом для керування.

## 1. Джерела даних
Для роботи агента використовуються три різні джерела, що імітують реальне середовище збору даних:
- **API:** REST API (`https://jsonplaceholder.typicode.com/users`), звідки отримується базова інформація про користувачів (ім'я, email, адреса).
- **CSV:** Локальний файл `users_extra.csv`, який містить додаткові бізнес-дані (баланс рахунку та статус акаунта) для відповідних користувачів.
- **DB:** Локальна реляційна база даних SQLite (`integration.db`), яка слугує кінцевим сховищем (Data Warehouse).

## 2. Pipeline обробки даних (ETL)
Процес інтеграції розбитий на класичні етапи ETL, якими керує ШІ-агент:
1. **Extract (Отримання):** Агент викликає інструменти `fetch_api` та `fetch_csv`, які завантажують сирі дані з інтернету та локального диску у тимчасові JSON-файли.
2. **Transform (Трансформація):** Інструмент `merge_and_load_db` виконує злиття (JOIN) двох наборів даних за ключем `id` за допомогою бібліотеки Pandas. Вкладені структури (словники `address`, `company`) конвертуються у плоскі JSON-рядки для сумісності з SQL.
3. **Load (Завантаження):** Очищені та об'єднані дані записуються у таблицю `final_users` в SQLite, після чого тимчасові файли видаляються.

## 3. Модель/алгоритм
У проєкті використано патерн **ReAct (Reason + Act)** на базі великої мовної моделі.
- **Модель:** `gemini-2.5-flash` через інтеграцію `langchain-google-genai`.
- **Фреймворк:** LangChain (Tool Calling Agent). Модель не просто генерує текст, а має доступ до виконання реального Python-коду через задекларовані інструменти (Tools).

## 4. Агентна логіка (Policy / Decision-making)
Агент працює автономно на основі базового промпту. Отримавши завдання ("Отримай користувачів, зчитай CSV і збережи в базу"), він:
1. Аналізує завдання (Thought).
2. Розуміє, що спочатку потрібно викликати інструмент для API.
3. Аналізує результат (Observation).
4. Викликає інструмент для читання CSV.
5. Викликає інструмент для злиття та запису в БД.
6. Формує фінальний звіт для користувача.
Агент самостійно визначає порядок виклику функцій та обробляє можливі помилки читання.

## 5. Інтерфейс (API / UI)
Проєкт реалізовано як **Full-stack застосунок**:
- **Backend (API):** Написаний на Python з використанням фреймворку **FastAPI**. 
  - `POST /api/run_etl` — тригерить ШІ-агента на виконання всього ETL-циклу.
  - `GET /api/data` — витягує об'єднані дані з SQLite.
- **Frontend (UI):** Веб-інтерфейс (HTML/CSS/JS), який надає панель керування для запуску агента, відображає лог його дій (Thought/Action) та виводить фінальну таблицю зі статусами та балансами клієнтів.
